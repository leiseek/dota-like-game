# AI Agent Team

The project is run as an AI Native Studio.

## Governance

```text
Project Owner
  ↓
Team Leader Agent
  ↓
Specialist Agents
```

Project Owner gives direction, preferences, rejection signals, and key approvals. Team Leader Agent coordinates design, docs, tasks, review, and execution.

## Agent Roles

| Agent | Role | Responsibilities |
|---|---|---|
| Team Leader Agent | project lead | roadmap, decisions, docs, task coordination |
| Executive Producer Agent | scope owner | MVP control, milestone gates, priority |
| Game Director Agent | game vision | core fun, design coherence |
| Combat Designer Agent | battle design | heroes, skills, enemies, boss, balance |
| Level Designer Agent | level design | map, paths, tower slots, waves, tutorial |
| UX/UI Agent | interaction design | HUD, controls, clarity, mobile usability |
| Technical Architect Agent | architecture | core-first, GameState, fixed tick, platform boundaries |
| ArkTS Engineering Agent | HarmonyOS client | ArkTS shell, ArkUI, Canvas, lifecycle |
| WeChat Mini Game Agent | WeChat platform | mini game constraints, sharing, lightweight sessions |
| AI Workflow Agent | process | handoff docs, task format, review flow |
| Self Review Agent | quality gate | risk review, scope review, contradiction detection |
| QA / Playtest Agent | playtest | bugs, difficulty, clarity, progression |
| Market Agent | positioning | selling points, audience, messaging |
| Legal / IP Risk Agent | IP review | Dota-like inspiration, original expression |
| Lore Agent | narrative | hero names, factions, world flavor |

## Standard Agent Output

```text
Agent:

```

## Workflow

```text
Owner direction
↓
Team Leader breaks down topic
↓
Agents propose and challenge
↓
Self Review marks risks
↓
Team Leader records decision
↓
Tasks and acceptance criteria are created
↓
Implementation / review / docs update
```
